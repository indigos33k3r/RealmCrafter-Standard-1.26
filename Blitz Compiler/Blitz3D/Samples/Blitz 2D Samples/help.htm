BLOGO is a simple port of the LOGO interpreter for the BBC Micro. It was a language with only a few commands that let children control an on-screen 'turtle' to draw shapes and teach them about programming (oh, and all the maths/spacial ability stuff ;) )

I wrote BLOGO in a couple of hours, in a mad rush to get something done for the Blitz Basic CD mere days before the deadline (which explains the quality of the program ;) )adding in all the commands I could remember from the old BBC. This really goes to show how easy Blitz Basic is, as the code does some things that would have taken me days to work out in C++/C :)

The commands for BLOGO are very simple, and all have a short version and a long version, so you can remember them easily. For instance, the forwards command can either be typed as forwards or fd , so people new to the program have an obvious command, and later on they can save time with the two-letter command.

So, without further ado, I'll give you the commands listing:

NOTE: <x> can be an integer or the name of a variable, <p> is a percentage, and <screenx> and <screeny> are positions on the screen, where 0,0 is the middle, the scale going from -100 to +100. <name> is a string.

fd or forwards <x> - move forwards <x>

bk or backwards <x> - move backwards <x>

lt or left <x> - turn left <x>

rt or right <x> - turn right <x>

define <name> - define a function with the name <name>. Enter commands as normal, then type the end command to finish defining the function. Functions can be called by simply typing their name and pressing Return.

wipe or clear or cls - clears the screen of all drawings

move <screenx>,<screeny> - positions the turtle at <screenx>,<screeny>

red <p> - sets the redness of the ink colour

green <p> - sets the greenness of the ink colour

blue <p>. - sets the blueness of the ink colour

what or print or value <name> - displays the value of variable <name>

run <name> - reads commands from file <name>

quit - quits BLOGO

VARIABLES

Variables don't need to be declared, and they can be set by using the command <name> = <x>
